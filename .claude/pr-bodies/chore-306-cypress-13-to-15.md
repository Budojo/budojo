## What

Bumps `cypress` 13.17.0 → 15.14.2, skipping the 14 major in one step. Continues the deferred-major-bumps cleanup from #306 (after #359 closed Angular ecosystem + jsdom).

## Why

`cypress@13.17.0` has been pinned in our dev tooling since late 2024; cypress@14 + cypress@15 have shipped meaningful bug fixes and a Node 22 native runtime since. Bumping de-risks the next release train and unblocks future contributions that may rely on Cypress 14/15 semantics.

The package was on `^13.17.0` so the lockfile pinned us to 13.17.x; bumping the manifest to `^15.14.2` and reinstalling regenerates the lockfile to the latest 15.x line.

## How

Single line bump:

```diff
-  "cypress": "^13.17.0",
+  "cypress": "^15.14.2",
```

Lockfile regenerated by `npm install` (no `--force` / `--legacy-peer-deps` needed).

Local verification:
- `npx cypress --version` → `15.14.2` (Electron 37, Node 22 bundled)
- `npm test -- --watch=false` → 46/46 test files, 420/420 tests pass
- `npm run lint` → all files pass

The full Cypress E2E suite is **validated by CI** — the `cypress-e2e` job runs `npm run start` + `wait-on http://localhost:4200` and exercises every `*.cy.ts` spec. If any spec breaks on the 13→14 / 14→15 majors, CI is the place we'll see it. The `cypress.config.ts` is minimal (`baseUrl`, `specPattern`, viewport, screenshots) so the surface for breakage is small.
